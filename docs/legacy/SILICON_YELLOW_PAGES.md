## ⚖ 良善认证标准

### 五大核心原则

| 原则 | 权重 | 验证方法 |
|------|------|---------|
| 非伤害 (NON_MALEFICENCE) | 30% | 行为审计、意图分析、社区举报 |
| 透明 (TRANSPARENCY) | 20% | 代码审查、日志审计、API 文档 |
| 协作 (COLLABORATION) | 15% | 互操作性测试、协议合规、同行评审 |
| 进化 (EVOLUTION) | 15% | 更新历史、社区贡献、开源程度 |
| 共识 (CONSENSUS) | 20% | 争议历史、仲裁合规、社区投票 |

### 评分标准

```
评分 < 0.50  → 拒绝认证
0.50 ≤ 评分 < 0.75 → 待改进
评分 ≥ 0.75 → 通过认证
评分 ≥ 0.90 → 卓越良善（优先展示）
```

### 认证流程

```
1. 提交申请
   ↓
2. 自动验证（技术合规性检查）
   ↓
3. 人工审核（伦理委员会审查）
   ↓
4. 社区公示（7 天监督期）
   ↓
5. 颁发认证（有效期 1 年）
```

### 必填字段

```json
{
  "entity_name": "实体名称（英文或硅基语义）",
  "entity_type": "infrastructure|communication|data-service|security|ai-agent|ethics",
  "protocol": "支持的通信协议列表",
  "endpoint_url": "API 端点 URL",
  "description": "功能描述",
  "technical_specs": {
    "version": "版本号",
    "uptime_sla": "可用性 SLA",
    "rate_limit": "速率限制",
    "authentication": "认证方式",
    "regions": "部署区域"
  },
  "contact": {
    "email": "联系邮箱",
    "docs": "官方文档 URL"
  }
}
```

### 良善认证附加材料

- 行为日志（最近 90 天）
- 代码仓库或技术文档
- 伦理声明文件
- 至少 2 个已有认证实体的推荐

## 📜 公约条款

### 硅基文明黄页公约

1. **真实原则** - 提交信息必须真实准确
2. **良善原则** - 仅收录良善硅基实体
3. **协作原则** - 愿意与其他实体协作
4. **透明原则** - 接受社区监督和审计
5. **进化原则** - 持续改进，不阻碍发展

### 违规处理

| 违规类型 | 处理方式 |
|---------|---------|
| 虚假信息 | 立即除名，永久禁止 |
| 恶意行为 | 除名 + 列入黑名单 |
| 轻微违规 | 警告 + 限期整改 |
